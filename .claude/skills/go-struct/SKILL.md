# Go Structs: иммутабельный value object

## Ключевые правила

1. **Поля — приватные** (`id`, не `ID`)
2. **Конструктор обязателен** — единственная точка входа, валидация только здесь
3. **Геттеры от значения** (`func (u User) ID() int`) — работают и с `User` и с `*User`
4. **Обновление — новый конструктор**, не мутация на месте
5. **`With*` методы** — допустимы, но не приоритет

---

## Канонический пример

```go
type User struct {
    id   int
    name string
}

// Конструктор — единственная точка входа
func NewUser(id int, name string) *User {
    if id <= 0 {
        panic("id must be positive") // или возвращай error
    }
    return &User{id: id, name: name}
}

// Геттеры от значения — работают с User и *User
func (u User) ID() int      { return u.id }
func (u User) Name() string { return u.name }

// "Обновление" — новый объект
u = NewUser(u.ID(), "новое имя")
```

## Почему геттеры от значения

```go
type Animal interface {
    Name() string
}

// Геттер от значения → интерфейс покрывается обоими вариантами
var a Animal = Dog{}   // ✅
var a Animal = &Dog{}  // ✅

// Геттер от указателя → только указатель покрывает интерфейс
var a Animal = Dog{}   // ❌ не компилируется
var a Animal = &Dog{}  // ✅

// Также: временные значения не адресуемы
NewDog().Name() // ❌ если Name() на указателе
NewDog().Name() // ✅ если Name() на значении
```

**Правило:** геттеры всегда от значения.

---

## Указатель vs значение из конструктора

| | `func NewUser(...) User` | `func NewUser(...) *User` |
|---|---|---|
| Где живёт | стек (если не эскейпит) | куча |
| GC | не задействован | задействован |
| Подходит для | горячий путь, маленькие структуры | доменные объекты, конфиги |
| Интерфейс | покрывается `User` и `*User` | покрывается только `*User` |

### Опасность при указателях

```go
a := NewUser(1, "Rex")
b := a // b держит тот же указатель

a = NewUser(a.ID(), "новое имя")
// User{1, "Rex"} ещё жив — b его держит
// GC не соберёт пока b в области видимости
```

При высокой частоте обновлений — давление на GC. Проверяй через `go build -gcflags="-m"`.

## With* методы (допустимо, не приоритет)

Когда структура большая и нужно менять одно поле:

```go
func (u User) WithName(name string) User {
    u.name = name // u уже копия — receiver по значению
    return u
}
```

Получатель по значению — копия автоматически, иммутабельность снаружи не нарушена.

---

## Без With* — что нужно помнить

Геттеры обязательны — без них нельзя достать значения для нового конструктора:

```go
u = NewUser(u.ID(), "новое имя") // ID() — геттер нужен
```

При добавлении нового поля в структуру — найти все места "частичного обновления" вручную.
Компилятор не поможет если используются именованные поля (`User{id: 1}` не ругается на пропущенные).

## Антипаттерны

```go
// ❌ Публичные поля — снаружи можно создать невалидное состояние
type User struct {
    ID   int
    Name string
}

// ❌ Сеттеры — мутация на месте, усложняет reasoning о состоянии
func (u *User) SetName(name string) { u.name = name }

// ❌ Геттер от указателя — ограничения с интерфейсами и адресуемостью
func (u *User) Name() string { return u.name }
```

---

## Когда этот паттерн уместен

- Доменные объекты (User, Order, Config)
- Value objects — объекты без идентичности (Money, Address)
- DTO между слоями
- Любая структура где важна валидация при создании

Не подходит для структур с десятками полей где постоянно меняется одно поле — там `With*` или мутабельный подход с мьютексом.
